# vulnerabilities.md

## Current Security Notes

- The CLI can modify files in the selected workspace. Path traversal is blocked by resolving paths and checking that they remain inside the workspace, but symlink-heavy projects should still be treated carefully.
- Plan Mode is intended as a read-only planning boundary: write/edit/shell tools are not exposed to the model and are also blocked in the agent executor. Treat this as a CLI safety guardrail, not a sandbox for arbitrary untrusted model output.
- Shell execution is disabled by default unless `permission.bash` is set to `allow` in `yacli.jsonc` or `--allow-shell` is passed. Commands run with the user's local permissions and can still be dangerous even after confirmation. File-write `--yes` does not approve shell commands; shell auto-approval requires the separate `--yes-shell` flag.
- Tool arguments come from an LLM. The code validates tool names and workspace paths, but every new tool should be treated as untrusted input until validated. The `edit_file` tool only performs exact text replacement inside a resolved workspace path and still requires file-write confirmation.
- File read/search tools reject binary files and files over the configured read limit, reducing accidental disclosure of large dumps, generated artifacts, or non-text data. The limit is a guardrail, not a data-classification system.
- Prompt `@file` references are resolved inside the workspace. Text references are copied into the model prompt, and supported image references (`jpg`, `jpeg`, `png`, `webp`, `gif`) are Base64-encoded and sent to the model API. Treat `@image` as an explicit data egress boundary; avoid attaching screenshots or images containing secrets.
- The fallback parser accepts legacy text tool calls such as `[TOOL_CALL_START]write {...}`. This is intentionally limited to known tool names, but malformed model output may still fail and should be logged rather than executed blindly.
- API credentials are read from `yacli.jsonc`. The local `yacli.jsonc` is intentionally ignored by `.gitignore`, but local malware, overly broad file permissions, shell history, screenshots, or accidental copy-paste can still expose IAM/API tokens. CLI warns when `yacli.jsonc` is accessible to group/others; prefer `chmod 600 yacli.jsonc`. Rotate the token if it is shared or leaked.
- Non-Yandex providers configured with `@ai-sdk/openai-compatible` or `@ai-sdk/anthropic` create additional data egress boundaries. Prompts, tool results, and `@file`/`@image` attachments are sent to the provider selected by `model`; review `baseURL` and model selection before attaching sensitive files.
- Do not bypass TLS verification to work around certificate errors. Install/update the Python certificate bundle instead, otherwise API credentials could be exposed to a machine-in-the-middle attack.
- Prompt history is saved as plain text in `~/.yandexgpt/history` by default. Do not paste secrets into prompts; delete or relocate the history file with `YANDEXGPT_HISTORY` if needed.
- Chat sessions are saved as JSON in `~/.yandexgpt/sessions` by default and may include prompts, assistant responses, tool results, and text `@file` attachments. Image Base64 payloads are stripped before saving, but image names and descriptions may remain. Do not paste or attach secrets unless you are comfortable storing them locally.
